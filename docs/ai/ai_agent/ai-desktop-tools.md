# 办公小浣熊 2.0、Everywhere——AI 桌面办公工具横向对比

> 除了 Claude Cowork、Kimi Work、WorkBuddy 这类全能 Agent 桌面工作台，还有一些定位更聚焦的 AI 桌面工具。它们不追求大而全，而是在"本地文件处理"或"屏幕感知"等特定场景中做得更深入。办公小浣熊 2.0 主打本地文件 + 企业 IM 闭环，Everywhere 主打无障碍 API 屏幕感知。

[[toc]]

---

## 一、AI 桌面工具的两个不同方向

通用 Agent 桌面工作台（如 Claude Cowork、WorkBuddy）追求的是全能——文档、代码、浏览器、数据源全覆盖。但还有另一类 AI 桌面工具，它们不试图覆盖所有场景，而是把某一个场景做透：

- **办公小浣熊 2.0（商汤）**：聚焦本地文件处理 + 企业协作 IM 打通，解决"AI 生成的报告还要手动复制到飞书/钉钉"的最后一公里
- **Everywhere（独立开发者）**：聚焦屏幕感知 + 无障碍 API，解决"看到报错还得复制粘贴到 AI 对话框"的上下文断裂问题

两者正好互补——一个管文件到输出的自动化闭环，一个管当前屏幕的即时 AI 辅助。

## 三、办公小浣熊 2.0——从本地文件到企业 IM 的闭环

### 3.1 产品背景

办公小浣熊是商汤科技推出的 AI 办公桌面 Agent。1.0 版本本质上是把网页版搬到桌面上，2.0 版本核心升级是从"等你提问"变为"帮你执行"。

### 3.2 六大核心能力

**① 本地文件直读**
授权一个目录后直接读取 Excel、CSV、PDF、Word、PPT 等 20+ 格式，无需手动上传下载。你可以说：帮我分析这个文件夹里的销售数据，找出增长最快和下滑最明显的品类。

**② 一句话操控浏览器**
内置 Playwright MCP，告诉它一个任务，它会自己打开网页、读取内容、抓取关键信息。

**③ Quick Bar 全局唤起**
任何应用内 ⌘K / Ctrl+K 唤出，选中文字直接翻译/总结/改写。Excel 里选中单元格直接分析异常、归类标签。

**④ 打通国内办公 IM**
桌面版 2.0 可以直接连接飞书（20+ Skill 内置），企微、钉钉正在路上。提供飞书文档空间 GUI 管理面板，不装飞书客户端也行。

**⑤ 本地记忆跨会话生效**
记住职业、行业、常用术语、输出偏好，越用越懂你。

**⑥ 定时任务自动执行**
设定好执行时间和任务内容（如每天早上 9 点读取最新数据，生成昨日业务日报），AI 自动完成。

### 3.3 与 WorkBuddy 的定位差异

| 维度 | 办公小浣熊 2.0 | WorkBuddy |
|:---|:---|:---|
| 核心优势 | 本地文件闭环 + 企业 IM 集成 | 生态完整（Skill 市场 + 多模型 + 专家功能） |
| 文件处理 | 授权目录直读 20+ 格式 | Word/PDF/PPT/Excel 全格式读写 |
| IM 集成 | 飞书已通，企微/钉钉开发中 | 通过 Claw 集成微信/企微/QQ 等 7 种平台 |
| 全局唤起 | ✅ Quick Bar | — |
| 专家/Skill | — | 181+ 专家 + Skill 市场 |

核心差异：办公小浣熊主打**本地文件 + 企业 IM 闭环**，适合需要从本地文件处理直接输出到企业协作平台的场景。WorkBuddy 赢在**生态完整**，适合个人办公 + 内容创作。

## 四、Everywhere——屏幕感知 AI，不切窗口的助手

### 4.1 产品背景

Everywhere（5.9k Star）由国内独立开发者 DearVa 主导，背后公司 Sylinko Inc.。技术栈 .NET 10 + Avalonia UI，目前 Windows 10+ 和 macOS 12+ 已稳定，Linux 开发中。

### 4.2 屏幕感知不是截图

Everywhere 与 ChatGPT 桌面版的本质差异在于屏幕感知方式：

```
ChatGPT 桌面版                        Everywhere
┌─────────────────────┐              ┌─────────────────────┐
│ 空白聊天框            │              │ 快捷键唤起            │
│ 你看的报错它看不到      │              │ 已读取当前屏幕内容      │
│ 你需要 Ctrl+C → Ctrl+V │              │ 直接提问即可          │
│ 思路断裂              │              │ 不切窗口              │
└─────────────────────┘              └─────────────────────┘
```

Everywhere 通过操作系统 **Accessibility API** 直接拿到当前窗口的结构化数据：你在用什么软件、当前是哪个输入框、选中了哪段文本。知道你在 IDEA 里、当前是 UserService.java 第 42 行、光标在哪个方法体里——这是截图模式做不到的。

常用软件（VS Code、Chrome、IDEA、Office）无障碍 API 完整，冷门软件回落到截图模式。

### 4.3 五个典型场景

1. **报错诊断不用截图**：终端窗口里直接唤起，问"这个报错是啥？"——它读取屏幕上的报错栈 + 项目结构给出诊断
2. **长技术文章一键摘要**：打开 80 行源码或 100 页 PDF，说"抓 5 个核心要点"
3. **自然语言调系统命令**：说"清掉 8080 的进程"——无需手打 lsof -i :8080
4. **英文文档随手翻译**：对着文本唤起，直接出译文
5. **Code Review 辅助**：看 PR 时问"这段改动主要在干啥？有什么风险？"

### 4.4 模型支持

OpenAI、Anthropic、Google Gemini、DeepSeek、Kimi、MiniMax、SiliconCloud、Ollama，以及任意兼容 OpenAI API 的端点。自带云服务——不想填任何 API Key，登录就能用。

Agent 能力：网页浏览、子代理派发、本地文件系统、终端脚本执行、MCP 工具协议。

## 五、选型建议

| 场景 | 推荐选择 |
|:---|:---|
| 日常办公，本地文件处理 + 企业 IM 协作 | **办公小浣熊 2.0** |
| 开发调试，看报错/读代码/翻译文档 | **Everywhere** |
| 需要定时自动跑报表、日报 | **办公小浣熊 2.0** |
| 追求操作流畅度，不想切窗口打断思路 | **Everywhere** |
| 需要三端协同（桌面+移动+网页） | **办公小浣熊 2.0** |
| 需要通用桌面工作台（全面能力） | 参考 [Claude Cowork、Kimi Work、WorkBuddy](./claude-cowork.md) |

---

## 六、总结

办公小浣熊 2.0 和 Everywhere 代表了 AI 桌面工具的两种极致：

- **办公小浣熊 2.0** 走的是"工作流闭环"路线——文件处理、IM 集成、定时任务，把 AI 嵌入到你的日常办公流程中
- **Everywhere** 走的是"上下文感知"路线——不让你离开当前窗口，在遇到问题的瞬间提供 AI 辅助

两者不互斥，甚至可以互补使用：用 Everywhere 做即时诊断和翻译，用办公小浣熊做批量文件处理和定期报表。而如果需要一个全能工作台，则回到通用 Agent 桌面产品。
